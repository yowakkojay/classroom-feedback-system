## 1. 系统简介

课堂学情监控管理系统是一款用于课堂教学的实时互动工具。系统支持三种角色：

| 角色 | 主要功能 |
|------|----------|
| **学生** | 提交课堂理解度反馈（表情评分）、参与课堂讨论 |
| **教师** | 管理课程与课堂、查看实时统计图表、历史查询与导出、参与讨论 |
| **管理员** | 管理系统用户（增删改查） |

系统地址：
- 前端页面：`http://localhost:5173`（开发环境）
- 后端 API：`http://localhost:8000`（开发环境）

## 2. 登录系统

所有角色使用同一个登录页面。

1. 打开系统首页，自动跳转到登录页
2. 输入**用户名**和**密码**
3. 点击**登录**按钮
4. 系统根据账号角色自动跳转到对应页面：
   - 学生 → 学生仪表盘
   - 教师 → 课堂管理页
   - 管理员 → 用户管理页

退出登录：点击页面右上角的**退出登录**按钮。

## 3. 学生端操作指南

### 3.1 查看活跃课堂

登录后进入学生仪表盘，页面以卡片形式展示当前所有**正在进行中的课堂**。每张卡片显示：
- 课程名称
- 开始时间
- 刷新间隔（时段长度）

### 3.2 提交表情反馈

在活跃课堂卡片中，点击表情按钮提交你对当前时段的理解程度：

| 评分 | 表情 | 含义 |
|------|------|------|
| 1 | 😫 | 完全不懂 |
| 2 | 😟 | 不太懂 |
| 3 | 😐 | 一般 |
| 4 | 🙂 | 基本理解 |
| 5 | 😊 | 理解 |
| 6 | 🤩 | 完全理解 |

- 每个时段可以提交一次反馈
- 系统会自动计算当前所处的时段
- 提交成功后会显示提示信息

### 3.3 参与课堂讨论

1. 在课堂卡片中点击**讨论区**按钮
2. 进入实时聊天界面，可以看到本课堂的所有讨论消息
3. 在底部输入框输入消息，按 **Enter** 或点击**发送**按钮发送
4. 消息实时推送，无需刷新页面
5. 每条消息显示发送者姓名、角色标签和发送时间

## 4. 教师端操作指南

教师端左侧有导航菜单，包含**课堂管理**和**历史查询**两个页面。

### 4.1 创建课程

1. 进入**课堂管理**页面
2. 在课程列表上方点击**新建课程**按钮
3. 在弹出对话框中输入课程名称
4. 点击**确定**，课程创建成功后出现在课程列表中

### 4.2 开始课堂

1. 在课程列表中，点击对应课程的**开始课堂**按钮
2. 在弹出对话框中设置**刷新间隔**（1-30 分钟），即每个时段的时长
3. 点击**确定**，课堂开始
4. 课堂开始后会出现在页面上方的**当前活跃课堂**区域

### 4.3 查看实时统计

1. 在活跃课堂区域，点击**查看统计**按钮
2. 进入统计页面，可以看到：

**关键指标卡片：**
- 参与人数 — 已提交反馈的学生数
- 平均分 — 当前时段的平均理解度
- 方差 — 反映学生理解度的分散程度
- 当前时段 — 正在进行的时段编号

**三种图表：**
- **柱状图** — 展示 1-6 分各分数段的学生人数分布
- **饼图** — 展示各分数段的占比
- **折线图** — 展示各时段平均分的变化趋势

统计数据通过 WebSocket 实时更新，同时每 30 秒自动刷新。

### 4.4 导出数据

在统计页面或历史查询页面，点击**导出 Excel**按钮，系统将当前课堂的反馈数据导出为 Excel 文件并自动下载。

### 4.5 结束课堂

在活跃课堂区域，点击**结束**按钮即可结束课堂。结束后学生将无法继续提交反馈。

### 4.6 历史查询

1. 点击左侧菜单进入**历史查询**页面
2. 可按以下条件筛选：
   - **课程名称** — 输入关键字搜索
   - **时间范围** — 选择起止日期
3. 点击**搜索**，下方表格显示匹配的历史课堂记录
4. 每条记录显示：课程名称、开始/结束时间、参与人数、平均分、方差
5. 可对每条记录点击**详细统计**查看图表，或点击**导出**下载 Excel

### 4.7 课堂讨论

1. 在活跃课堂区域点击**讨论区**按钮
2. 功能与学生端讨论区相同，但教师额外支持**上传图片**
3. 点击输入框旁的上传按钮选择图片，可同时附带文字消息

## 5. 管理员端操作指南

管理员登录后进入**用户管理**页面。

### 5.1 查看用户列表

- 页面以表格形式展示所有系统用户
- 显示字段：ID、用户名、姓名、角色、联系方式、性别、年龄、学号
- 角色以彩色标签区分：学生（灰色）、教师（橙色）、管理员（红色）
- 每页显示 20 条记录，超出后显示分页器

### 5.2 搜索与筛选

- **关键字搜索** — 在搜索框输入用户名、姓名或手机号，按 Enter 或点击搜索
- **角色筛选** — 在下拉框中选择学生/教师/管理员进行过滤
- 点击搜索框的清除按钮可重置搜索条件

### 5.3 添加用户

1. 点击**添加用户**按钮
2. 在弹出表单中填写用户信息：

**必填项：**

| 字段 | 说明 |
|------|------|
| 用户名 | 系统登录名，不可重复 |
| 密码 | **至少 6 位** |
| 角色 | 选择学生、教师或管理员 |

**选填项：** 姓、名、性别、年龄、联系方式

**角色特定字段：**
- 选择**学生**角色时，额外显示学生信息栏：
  - **学号**（必填） — 学生的唯一标识
  - 年级、班级、专业（选填）
- 选择**教师**或**管理员**角色时，额外显示教师信息栏：
  - 职称（选填）

3. 点击**保存**提交

### 5.4 编辑用户

1. 在用户列表中点击对应行的**编辑**按钮
2. 弹出编辑表单，预填当前用户信息
3. 修改需要变更的字段
4. **密码**栏留空表示不修改密码；如需修改，输入新密码（至少 6 位）
5. 用户名在编辑模式下不可修改
6. 点击**保存**提交

### 5.5 删除用户

1. 在用户列表中点击对应行的**删除**按钮
2. 弹出确认对话框，确认后删除该用户
3. 删除操作不可撤销，请谨慎操作

## 7. 常见问题

### Q: 登录时提示"用户名或密码错误"？
确认用户名和密码是否正确。密码区分大小写。如忘记密码，请联系管理员重置。

### Q: 学生提交反馈后没有反应？
检查是否有正在进行中的课堂。反馈只能在活跃课堂的有效时段内提交。

### Q: 教师看不到统计数据？
确认课堂已经开始且有学生提交了反馈。如果刚开始课堂，需要等待学生提交后数据才会出现。

### Q: 讨论区消息不实时更新？
实时消息依赖 WebSocket 连接。确认后端使用了 ASGI 服务器（daphne）启动，而非普通的 `runserver`。启动命令：
```bash
daphne -b 0.0.0.0 -p 8000 config.asgi:application
```

### Q: 添加用户时提示验证错误？
请检查以下必填项：
- 用户名不能为空
- 密码至少 6 位
- 选择学生角色时必须填写学号
- 用户名不能与已有用户重复

### Q: 导出 Excel 没有反应？
确认浏览器没有拦截弹出窗口或文件下载。导出功能会直接下载 `.xlsx` 文件。

### Q: 页面显示"登录已过期"？
JWT 令牌有过期时间，过期后需要重新登录。这是正常的安全机制。
